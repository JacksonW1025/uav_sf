# M2b-1 Adversarial Shared-State Shim

M2b-1 adds the missing adversarial state-pollution capability for Phase 1. The shim modifies shared uORB estimator/control-state topics before publication, so classical Offboard and RAPTOR consume the same polluted state. This is the opposite of a RAPTOR-only observation edit and is the fairness boundary for this pass.

Confirmed primary bug count remains `0` in the runs completed here. This is not a robustness conclusion. It means this M2b-1 slice still needs a larger campaign and additional profiles before the Phase 1 search space is covered.

## Shim Mechanism

The shim is parameter-gated and active only when `M2B_EN=1` and the simulation time is within `M2B_START` to `M2B_END`.

| shared channel | publish point | profiles |
|---|---|---|
| `vehicle_local_position` velocity | `src/modules/ekf2/EKF2Selector.cpp` before publication | transport delay, bias, noise, NaN, Inf |
| `vehicle_attitude` quaternion | `src/modules/ekf2/EKF2Selector.cpp` before publication | transport delay, roll/pitch/yaw bias, noise, NaN, Inf |
| `vehicle_angular_velocity` xyz | `src/modules/sensors/vehicle_angular_velocity/VehicleAngularVelocity.cpp` before publication | transport delay, bias, noise, NaN, Inf |

Delay uses a 128-sample ring buffer and republishes old values while preserving current message timestamps. Bias and noise are applied axis-wise. NaN/Inf profiles are input-robustness probes, not realistic differential flight faults by themselves.

The user-facing parameter family is:

```text
M2B_EN, M2B_START, M2B_END, M2B_SEED
M2B_V_PROF, M2B_V_DLY, M2B_V_X, M2B_V_Y, M2B_V_Z
M2B_G_PROF, M2B_G_DLY, M2B_G_X, M2B_G_Y, M2B_G_Z
M2B_A_PROF, M2B_A_DLY, M2B_A_R, M2B_A_P, M2B_A_Y
```

Profile IDs are `0=off`, `1=delay`, `2=bias`, `3=noise`, `4=nan`, `5=inf`.

## Mitigation Switch

The paper's S2 accelerometer-integration IIR mitigation was searched for in the local PX4 `mc_raptor` module at `3042f906`. No `vehicle_acceleration` or `sensor_accel` subscription, no S2 IIR code path, and no RAPTOR parameter for enabling/disabling that mitigation were found. The only RAPTOR module parameters in this tree are the existing `MC_RAPTOR_*` controls plus the M2b shim parameters added by this patch.

Therefore the velocity-delay runs below measure the default local `mc_raptor` implementation without an identifiable accel-IIR mitigation switch. An on/off mitigation comparison could not be isolated in this commit.

## Fairness Evidence

For active shim profiles, `m2_5_estimator_fairness.py` now checks three conditions:

1. The `M2B_*` parameters match between classical and RAPTOR runs and match theta.
2. The target shared topic is polluted in both ULOGs relative to its groundtruth topic where available.
3. The corresponding `raptor_input` field carries the polluted signal during RAPTOR active time.

Representative evidence: `docs/m2b_velocity_delay_verify_4x_20260624/evals/m2b_velocity_delay_verify_4x_20260624_vdelay_030ms/m2b_1_fairness_m2b_velocity_delay_verify_4x_20260624_vdelay_030ms.json` reports `fair_shared_state_shim_pollution=true`, `state_shim_topic_polluted_both_runs=true`, and `state_shim_raptor_input_touch_verified=true`.

## State MAP-Elites

The M2b search script covers:

- Channels: velocity, angular velocity, attitude.
- Profiles: delay, bias, noise.
- Conjunction: optional secondary channel/profile.
- Regime: high TWR plus mild/high-frequency sine tracking.
- Fitness: continuous RAPTOR-minus-classical degradation above the configured 4x floor, with classical required usable for primary candidacy.
