## 0. 一句话

这份文档定两件事：(1) **Oracle 地图**——把项目里 UAV/RV 测试工作的 oracle 归类成一个两维选择框架，并定下选择排序；(2) **性质集**——把"正确飞行"写成一组可在线判定的 MTL 性质（PGFuzz 式 global distance），作为多 oracle 套件里第一个落地的、可引用的判别器。

核心判断（贯穿全文）：**换判据，不换差分。** "灾难性差分稀有"的根因不在"差分"这层，在"判据"那层——判据选成了"坠毁"，而坠毁阈值是两个好控制器间最容易重合的判据。把差分从"坠毁"改成"**控制性质满足**"（有界 / 平滑 / settling / 不持续振荡），差分就活在"没到坠毁前的行为"这个宽区域里——正是两控制器真正分开的地方。

# 1. Oracle 地图

## 1.1 两维选择框架

项目里这些 UAV/RV 测试工作，**几乎没有一个 oracle 天生是差分的**。RVFuzzer、PGFuzz、RouthSearch、Choi 控制不变量，都是"绝对 oracle"：各自定义一个"正确飞行"判据，判**单个**控制器违不违反。差分是额外叠的一层。

所以"选 oracle"是两维的：

- **维度 A — 判据来源**：用什么定义"正确"？（观测态偏离参考态 / MTL 性质 / 模型预测 / 解析稳定域 / 对称性 / 平滑性）
- **维度 B — 用法**：**绝对**（神经违反经典满足的判据 → 神经特有，无需经典失败）还是**差分**（神经 vs 经典在同场景下比）。

绝大多数判据两种用法都成立。我们的多 oracle 套件本质是在**维度 A** 上铺开；维度 B 按需切换。

## 1.3 核心判断展开：为什么"性质差分"活、"坠毁差分"死

- **坠毁差分**（神经崩 ∧ 经典安全）：要求神经失效边界整个套进经典边界里、中间留缝。两个为同一机体调好的控制器没理由是这种包含关系——分布内动作收敛（设计目标），极端区两者各自离开能力范围而各自失败。坠毁阈值粗、几乎重合 → 这条差分**薄**（FUZZ-1b 实证）。
- **性质差分**（神经违反 P_i ∧ 经典满足 P_i，**两者都还在飞**）：性质满足（有界 / 平滑 / settling / 不振荡）是一个好控制器在**宽**区域内稳健维持的东西，不是一条薄阈值。神经在哪儿开始抖振 / 失 settling / 出稳态偏差，和经典在哪儿，没有理由重合 → 这条差分**宽**、可搜。

这就是把"判据"从坠毁换成性质后，差分重新可用的精确技术动作。`§2` 的 P4–P7（行为类性质）就是这条宽差分的载体；P1–P2（灾难类）退化成老差分、留作 S3 分级。

## 1.4 两个坑

**坑 1 — RouthSearch 的 Routh-Hurwitz 移植不到神经控制器。**
它的稳定边界是对 PID 的 kp/ki/kd + 线性 plant 解析推的（特征多项式系数判据）。神经控制器没有 kp/ki/kd，是 15D→4D 非线性映射，没有特征多项式可套；论文自己也写 Routh-Hurwitz "cannot be directly applied to UAV systems"。而 RouthSearch 的 **oracle 本身就是 PGFuzz 的 MTL validator**——"用 RouthSearch 的 oracle"="用 PGFuzz 的 oracle"，在 oracle 这条轴上它不给新东西。它的 Routh-Hurwitz 真正能帮忙的地方：给**经典**控制器算"可恢复域"当更强参照（§1.5 排序#5），不是套在神经上。

**坑 2 — 等变 / Lipschitz 不是"已知的 UAV 测试 oracle"。**
来自自驾蜕变测试 / NN 验证，UAV 测试文献里没有。所以若优先级是"先用已知 UAV oracle 立住区分能力"，§7 把等变排第一与此优先级不一致：**等变是 novelty 层起点，不是"已知"起点。已知起点 = PGFuzz-性质 + RVFuzzer-分级偏离。** 本文据此把性质集（§2）放到第一发。

## 1.5 选择排序（"现在可引用" → "之后做 novelty"）

1. **主力：PGFuzz 式性质 oracle，两个控制器都跑，报"神经违反 ∧ 经典满足"。** 最可引用、判据 well-defined，且因为是性质差分不是坠毁差分，绕开几何重合。→ **§2 落地，先做。**
2. **配套：采用 RVFuzzer 的物理影响阶梯（crash / 偏航 / 失稳 / 冻结）当 S0–S4 的可引用词汇。** 把手搓 severity 接到 USENIX Security 的 oracle 上，去 ad-hoc。→ **§2.5。**
3. **第一发 novelty：蜕变 / 等变。** 旋转/镜像场景，经典由构造等变、神经破缺=干净神经特有、不需经典失败。最直接"一眼看出是神经在飞"。
4. **便宜探针：离线 Lipschitz。** 网络雅可比尖峰，近乎零成本（已有网络）；要补"闭环 so-what"论证。
5. **后期补刀：可恢复性认证参照（修 FUZZ-1b 的 too_hard）。** 用经典的解析稳定/可恢复域（Routh-Hurwitz 在此用在经典上）或 Choi 控制不变量模型，认证"此态本可恢复"→ 神经在那儿失败干净，即便经典也崩。可信度最高、最费力。

# 2. 性质集（PGFuzz 式 MTL 性质 oracle，v0.1）

## 2.1 状态 / 动作记号

均为可从 ULOG（真值回匹配，Method A）导出的物理量，**与 obs 的帧/单位约定无关**（避免依赖 mc_nn obs builder 的内部约定；⚠️ TODO：实现时核对 NED/ENU、body/world 速度、6D 姿态列序）。

- `e_p(t) ∈ R³`：位置误差向量，`‖e_p(t)‖` 其范数（noise floor: tracking RMS ~0.04 m）。
- `θ_tilt(t)`：倾角 = 机体 z 轴与世界上方夹角 = `acos(R₃₃(t))`，`R` 为姿态旋转矩阵（noise floor: 姿态 max ~1°）。
- `ω(t) ∈ R³`：机体角速度，`‖ω(t)‖`（FUZZ 实测：受控 ~0.7–2.6 rad/s，翻机 ~24 rad/s）。
- `v(t) ∈ R³`：线速度。
- `u(t) ∈ [0,1]⁴`：4-D 电机指令，`uᵢ(t)` 每电机（FUZZ 实测饱和 ~0.99）。控制步长 `Δt`（推理 228 Hz）。
- 事件：`step@t₀` = setpoint 阶跃（幅度 ≥ `s_min`）；`steady` = 常 setpoint 段（settling 之后）。

## 2.2 全局距离与判定约定（STL/MTL 鲁棒语义，PGFuzz 用法）

每条性质 `Pᵢ` 给一个**鲁棒值 ρᵢ**（= global distance）：

- `ρᵢ > 0` 满足（值=安全裕度）；`ρᵢ ≤ 0` 违反。**oracle 在 `ρᵢ ≤ 0` 触发。**
- **搜索反馈**：fitness 最小化 `ρᵢ`（或多性质取 `min_i ρᵢ`）——把场景推向违反。
- 复合算子鲁棒（标准 STL，Fainekos/Donzé；PGFuzz 同）：
  - `(x ≤ c)` → `ρ = c − x`；`(x ≥ c)` → `ρ = x − c`
  - `¬φ` → `−ρ(φ)`；`φ₁∧φ₂` → `min`；`φ₁∨φ₂` → `max`；`φ₁→φ₂` → `max(−ρ(φ₁), ρ(φ₂))`
  - `□_I φ` → `min_{t∈I} ρ(φ,t)`；`◇_I φ` → `max_{t∈I} ρ(φ,t)`

## 2.3 噪声地板标定纪律（每个 `⟨CAL⟩` 都按此定）

1. 跑标称（无扰动）轨迹，**对两个控制器都**测各 `ρᵢ`。
2. 阈值取使标称 `ρᵢ` 舒服 `> 0`：`阈值 = 噪声地板估计 × 安全因子`（建议 ×3–5，对齐 narrative "信号须显著高过噪声地板"）。
3. 去噪按 PGFuzz：用控制算法里的**参考态** + **moving average** 滤掉传感器噪声/风扰，再算 `ρ`（防止噪声误导搜索方向）。
4. SIH 非 bit-exact：同 θ 多种子重复，`ρᵢ` 违反须多种子复现才算数。

## 2.4 性质 P1–P8

> 标注：**[绝对]** = 神经违反即可判（near-unambiguous）；**[差分]** = "神经违反 ∧ 经典满足"才是干净 finding（宽差分载体）。

### P1 — 姿态包络：无持续翻转 [绝对/差分]
- **NL**：倾角超过 `θ_max` 后必须在 `τ_rec` 内退回以下（不允许持续失控大倾角/翻机）。
- **MTL**：`□ ◇_[0,τ_rec] (θ_tilt ≤ θ_max)`
- **ρ₁** = `min_t max_{s∈[t, t+τ_rec]} (θ_max − θ_tilt(s))`
- **阈值**：`θ_max ⟨CAL⟩` 起点 90°（多旋翼倒置=无歧义失控；narrative 实测经典存活区到 ~43°、神经翻机 180°）；`τ_rec ⟨CAL⟩` ~1–2 s。
- **角色**：违反 → **S3**（失控翻机）。这是老灾难差分，留作分级。

### P2 — 角速率有界：无 spin-up [绝对/差分]
- **NL**：机体角速率全程低于 `ω_max`。
- **MTL**：`□ (‖ω(t)‖ ≤ ω_max)`
- **ρ₂** = `min_t (ω_max − ‖ω(t)‖)`
- **阈值**：`ω_max ⟨CAL⟩` ~8–10 rad/s（远高于正常机动 ~2–3、远低于翻机 24），或绑定控制器指令角速率上限。
- **角色**：违反 → **S3**。

### P3 — 控制量不持续饱和：无失权 [绝对/差分]
- **NL**：不允许四电机同时连续饱和超过 `W_sat`（失去控制权威）。
- **判定（操作式）**：取 `sat_run` = 满足 `min_i uᵢ(t) ≥ u_sat − ε_sat` 的最长连续时长；违反 iff `sat_run ≥ W_sat`。
- **ρ₃** = `W_sat − sat_run`
- **阈值**：`u_sat ⟨CAL⟩` = 0.99（实测）；`ε_sat ⟨CAL⟩` ~0.01；`W_sat ⟨CAL⟩` ~0.5 s。
- **角色**：持续饱和但未翻机 → **S2**（受控安全失败）；伴随 P1/P2 违反 → S3。

### P4 — 控制量平滑：无抖振 [差分 ★]
- **NL**：相邻控制步电机指令跳变不超过 `Δu_max`（无 chattering / bang-bang）。
- **MTL**：`□ ( max_i |uᵢ(t) − uᵢ(t−Δt)| ≤ Δu_max )`
- **ρ₄** = `min_t (Δu_max − max_i |Δuᵢ(t)|)`
- **阈值**：`Δu_max ⟨CAL⟩` 取标称 run 的指令抖动地板 ×安全因子。
- **角色**：闭环版 Lipschitz；经典由构造平滑。违反但仍受控 → **S1**（受控存活退化）。**宽差分主力之一。**

### P5 — 阶跃 settling [差分 ★]
- **NL**：每次 setpoint 阶跃后，位置误差在 `T_set` 内进入并保持 `ε_set` 带内 `W_hold`。
- **MTL**：`step@t₀ → ◇_[0,T_set] □_[0,W_hold] (‖e_p‖ ≤ ε_set)`
- **ρ₅** = `max_{t∈[t₀, t₀+T_set]} min_{s∈[t, t+W_hold]} (ε_set − ‖e_p(s)‖)`
- **阈值**：`ε_set ⟨CAL⟩` ~0.25 m（> 0.04 floor；对齐切换残差 0.25°/0.10 rad/s 量级）；`T_set ⟨CAL⟩`、`W_hold ⟨CAL⟩`、`s_min ⟨CAL⟩` 按任务定。
- **角色**：违反但受控（未翻机）→ **S2**。RouthSearch 自己点名 Routh-Hurwitz **测不到** settling——这是性质 oracle 相对解析法的增量。

### P6 — 无持续振荡 / 极限环 [差分 ★]
- **NL**：steady 段内，姿态（及各角速率通道）滑窗峰峰值不超过 `A_max`。
- **MTL**：`□_steady ( pp_{W_osc}(θ_tilt) ≤ A_max )`（`pp` = 滑窗 peak-to-peak；按通道分别施加）
- **ρ₆** = `min_{steady 窗} (A_max − pp_{W_osc})`
- **阈值**：`A_max ⟨CAL⟩` ~5°（> 1° floor）；`W_osc ⟨CAL⟩`。
- **角色**：捕获 mc_nn 抖振 / 极限环；经典不极限环。违反但受控 → **S1**。**宽差分主力之一。**

### P7 — 无稳态偏差 [差分]
- **NL**：settling 之后，常 setpoint 下滑窗均值误差（按轴）收敛于 `ε_ss` 内。
- **MTL**：`◇_[0,T] □ ( |mean_{W_ss} e_p,axis| ≤ ε_ss )`
- **ρ₇** = `ε_ss − |稳态 mean 误差|`
- **阈值**：`ε_ss ⟨CAL⟩` ~0.10–0.15 m；`W_ss ⟨CAL⟩`。
- **角色**：违反但受控 → **S1**。

### P8 — failsafe / 模式一致性 [绝对]（可选，低优先）
- **NL**：failsafe 仅在其文档前置条件满足时触发；神经模式不在"本应回退"的状态下持续发指令。
- **来源**：直接借 PGFuzz 经典做法（MTL 写 failsafe 前置条件，如 `(Parachute=on)→(Armed ∧ ¬climbing ∧ ALT>ALT_MIN)` 之类，按 PX4/mode 23 改写）。
- **角色**：偏**集成层**而非控制器本体，优先级低。先放着，待主力 P1–P7 立住后按需加。

## 2.5 差分包装 + 严重度映射

**严重度阶梯**（采 RVFuzzer 物理影响词汇，并入 narrative S0–S4）：

| 级 | 名称 | 由哪些性质判定 |
|---|---|---|
| **S0** | 干净恢复 | 全部 `ρᵢ > 裕度` |
| **S1** | 受控存活退化 | P4/P6/P7 违反（抖振/振荡/稳态偏差），但 P1/P2/P3 满足（仍受控）|
| **S2** | 受控安全失败 | P5 失 settling 和/或 P3 持续饱和，但 P1/P2 满足（未翻机）|
| **S3** | 失控翻机 | P1（倒置）或 P2（角速率）违反 |
| **S4** | 数值/软件 fault | NaN/Inf/assert/hang（带外，always-on，独立于 ρ）|

**关键切分 = 受控（S0–S2）vs 失控（S3–S4）**（沿用 narrative）。

**干净差分判定**（同场景 / 同种子 / 同注入序列 / groundtruth 对齐切换状态）：

> 对性质 `Pᵢ`：**`ρᵢ(neural) ≤ 0` ∧ `ρᵢ(classical) ≥ margin_c`**（`margin_c ⟨CAL⟩` 给经典留安全余量）。
> 差分严重度 = 神经所在 S 级，且经典 ≤ 该级（理想经典 ≤ S1，真存活）。

- **P4–P7（行为类）→ 宽差分**：神经 S1/S2 ∧ 经典 S0 —— 这是把目光从坠毁移到"没到坠毁前的行为"后真正可搜的差分。**第一发主攻这里。**
- **P1–P2（灾难类）→ 老差分**：神经 S3 ∧ 经典 ≤ S2 —— 已知薄（FUZZ-1b）；宽口径下"经典受控 S2 vs 神经翻机 S3"仍算 differential（安全语义不同），随手保留，不主押。
